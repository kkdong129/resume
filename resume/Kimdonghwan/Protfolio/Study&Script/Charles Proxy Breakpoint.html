Copy of Charles Proxy의 Breakpoint 사용해보기

이 페이지에서는 Charles Proxy에서 제공하는 기능 중 Breakpoint을 사용해보겠습니다.

기능을 사용하기전에 간단히 개념을 알아보면, Breakpoint란 프로그램을 의도적으로 멈추게하는 지점을 말합니다.

프로그램을 잠시 멈추었다가 원하는 환경과 조건을 세팅한 후, 다시 실행했을때 프로그램의 기대 동작을 확인하는 개념입니다.

Charles에서는 Breakpoint 기능을 사용하여 호출 시점에 request와 response를 편집할 수 있습니다.

  • Charles를 실행하고 테스트 기기를 연결합니다.
  • 잘 연결이 되었는지 Structure와 Sequence를 확인합니다.
  • 어떤 api를 호출하는지 확인하기위해서 망고플레이트앱에서 키워드를 검색하고 결과를 확인합니다.
  • Charles로 돌아와서 Sequence를 다시 확인해봅니다.
  • 앱에서 키워드를 검색하면 stage.mangoplate.com/api/v8/search/by_keyword.json을 호출하여 검색했던 키워드(2021망고플레이트인기맛집)와 검색 결과를 JSON으로 확인할 수 있습니다.
  • 이제 Breakpoint 기능을 사용해서 결과로 나왔던 name(패스트라부티크)을 수정해보겠습니다.
  • 해당 api를 우클릭하여 Breakpoints를 체크합니다.
  • 상단메뉴 → Proxy → Breakpoints settings → 추가했던 api가 호출될때마다 Breakpoint가 될 수 있도록 Enabled Breakpoints를 체크해줍니다.
  • 망고플레이트 앱으로 돌아가서 다시 키워드를 검색합니다.
  • Charles로 돌아가면 Breakpoint 탭이 나오게됩니다. Edit Response 탭으로 이동합니다.
  • 하단에 JSON Text나 JSON에서 변경하고자하는 필드의 결과 값을 수정합니다. 저는 name을 김밥천국으로 수정하겠습니다.
  • 수정 후, 하단의 Execute를 클릭하고 망고플레이트 앱으로 돌아가서 결과를 확인합니다.

Response가 수정한대로 성공적으로 반영되었습니다. 🎉

이 페이지에서는 Breakpoint 기능으로 특정 host 또는 api 호출 시점에 Request와 Response를 수정해보았습니다. 앞으로 업무의 활용 방법은 고민해봐야합니다.
